# DebTracker

Веб-приложение для учёта должников в розничном магазине. Работает на телефоне и компьютере, данные синхронизируются через Google Sheets.

**Демо:** [dreamo-pg.github.io/debt-tracker](https://dreamo-pg.github.io/debt-tracker/)

## Функционал

### Управление долгами
- Запись нового долга с привязкой к клиенту, суммой, датой возврата и комментарием
- Частичные и полные оплаты с историей платежей
- Автоматическое определение статусов: открыт, просрочен, закрыт
- Прогресс-бар оплаты для каждого долга

### Клиенты
- База клиентов с именем и телефоном
- Автоподстановка имени при записи долга (datalist)
- Маска телефона: автоформат в `+7 (XXX) XXX-XX-XX`
- Рейтинг надёжности (5 звёзд): падает при просрочках, растёт при закрытии долгов
- Предупреждение о рисковых клиентах (рейтинг <= 2) при записи нового долга

### Главная
- Список просроченных долгов
- Ближайшие сроки возврата (7 дней)
- Оповещения о рисковых клиентах
- Бейдж с количеством просроченных долгов

### Аналитика
- Фильтр по месяцу и году
- Выдано за период (сумма и количество долгов)
- Возвращено за период (сумма и количество оплат)
- Количество должников за период
- Переходы долгов из прошлых периодов с расшифровкой по клиентам

### Журнал действий (только админ)
- Лог всех операций: кто, когда, что сделал
- Фильтры по пользователю и типу действия
- Хранится до 500 последних записей

### Система доступа
- 3 учётные записи: Админ (PIN 1111), Продавец 1 (PIN 2222), Продавец 2 (PIN 3333)
- Роли: админ видит журнал, настройки облака, может удалять записи; продавцы — только ввод и просмотр
- Автовход при повторном открытии

### Синхронизация и безопасность данных
- Автоматическая синхронизация с Google Sheets через Apps Script
- Merge-стратегия при конфликтах (два пользователя одновременно — данные объединяются, не перезаписываются)
- Локальный кэш в localStorage (работает без интернета)
- Экспорт/импорт данных в JSON (резервная копия)

## Технологии

- Vanilla HTML/CSS/JS — один файл `index.html`, без фреймворков
- Google Apps Script — REST API для чтения/записи в Google Sheets
- GitHub Pages — хостинг
- localStorage — офлайн-хранилище

## Структура

```
index.html              — приложение (фронтенд)
google_apps_script.js   — код для Google Apps Script (бэкенд)
```

## Настройка Google Sheets (одноразово)

1. Создайте новую Google Таблицу на [sheets.google.com](https://sheets.google.com)
2. Откройте **Расширения -> Apps Script**
3. Удалите весь код и вставьте содержимое `google_apps_script.js`
4. **Развернуть -> Новое развёртывание**
5. Тип: **Веб-приложение**, доступ: **Все**
6. При первом запуске разрешите доступ к таблице
7. Скопируйте URL и вставьте в константу `CLOUD_URL` в `index.html`

После развёртывания в таблице автоматически создадутся листы:
- **Клиенты** — id, name, phone, createdAt
- **Долги** — id, clientId, amount, createdAt, dueDate, comment
- **Платежи** — id, debtId, amount, date, comment
- **Журнал** — user, userName, action, detail, timestamp
- **Мета** — счётчики nextClientId, nextDebtId, nextPayId

## Локальная разработка

```bash
# Запуск dev-сервера
python3 -m http.server 8080

# Открыть в браузере
open http://localhost:8080
```

## Деплой

```bash
# Коммит и пуш на GitHub Pages
git add index.html google_apps_script.js
git commit -m "описание изменений"
git push origin main
```

Сайт обновляется автоматически через 1-2 минуты после пуша.

## Переразвёртывание Apps Script

При изменении `google_apps_script.js`:

1. Apps Script -> вставить новый код
2. **Развернуть -> Управление развёртываниями -> карандаш**
3. Версия: **Новая версия**
4. **Развернуть**

Если URL не изменился — обновлять `CLOUD_URL` не нужно.
